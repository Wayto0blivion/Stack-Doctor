Reports Docker Compose networking and configuration mistakes directly in the compose file — the same checks shown in the Stack Doctor tool window, anchored to the exact line they concern.

Findings include depends_on targets that don't exist, dependency cycles, host port conflicts, ports bound to 127.0.0.1 (unreachable from other devices), services depended on without a healthcheck, undeclared named volumes and networks, missing bind-mount paths and env_files, and reverse proxies that publish no ports.

Quick-fixes are offered where one is unambiguous: publishing a loopback-bound port on all interfaces, and declaring a missing named volume or network under the top-level section.
